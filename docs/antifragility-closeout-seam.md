# Antifragility Closeout Seam

## Purpose

A reviewed closeout should be able to carry stress evidence across the workspace without inventing a new authority center.

## What a reviewed stress bundle can contain

A bounded bundle may include:

- source receipt refs
- runtime degradation receipt refs
- routing hint refs
- memo failure-lesson refs
- bounded eval report refs
- a reviewed next-step brief

## What the closeout seam should do

A healthy closeout seam should let operators or wrappers:

- submit a reviewed artifact plus stress evidence
- build a manifest that keeps refs legible
- process the inbox without pretending absent evidence exists
- preserve audit-only closeout when owner-local receipts are not available yet

## What it should not do

- auto-promote stress bundles into memo
- auto-publish stats claims
- auto-trigger runtime repair
- treat routing hints as source truth

## Suggested touchpoints

Wave 2 should fit naturally with existing CLI surfaces such as:

- `aoa skills enter`
- `aoa skills guard`
- `aoa closeout submit-reviewed`
- `aoa closeout build-manifest`
- `aoa closeout run`
- `aoa closeout process-inbox`

## Minimal contract idea

A reviewed stress closeout should make it easy to answer:

- what stress evidence was active
- what remained blocked
- what reviewed steps were taken
- what next step is recommended
- what still needs explicit owner-local proof later
