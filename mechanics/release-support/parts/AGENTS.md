# AGENTS.md

## Release Support Parts Route

Use these parts when the SDK validates, audits, publishes, or publicly frames
release claims without becoming GitHub, package-index, or sibling-release
truth.

Active part homes:

- `release-audit-publish-helper/` keeps release audit and publish helper
  behavior bounded to checked release surfaces.
- `public-support-ci-posture/` keeps public support, release claim, and CI tier
  posture explicit and short.

Keep publication truth in tags, GitHub Releases, package indexes, and owner
repositories. SDK release-support parts may verify and route claims; they do
not invent release state.
