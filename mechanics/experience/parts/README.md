# Experience Parts

Functioning Experience SDK parts live here. Each active part provides a local
operation card, contract, validation route, and artifact homes.

## Active Parts

- [capture-pipeline-helper](capture-pipeline-helper/README.md) - keeps the
  capture/pipeline helper seam as an SDK-owned contract without turning
  Experience capture into runtime activation.
- [adoption-federation-helper-contracts](adoption-federation-helper-contracts/README.md) -
  keeps adoption, federation, harvest, pattern, KAG, and ToS dossier helper
  payloads in a bounded SDK contract route.
- [deployment-watchtower-helper-contracts](deployment-watchtower-helper-contracts/README.md) -
  keeps deployment, certification, regression, rollback, release-candidate, and
  watchtower helper payloads in a bounded SDK contract route.
- [governance-runtime-helper-contracts](governance-runtime-helper-contracts/README.md) -
  keeps council, appeal, veto, constitution-runtime, queue, sealing, and replay
  helper payloads in a bounded SDK contract route.
- [office-release-train-helper-contracts](office-release-train-helper-contracts/README.md) -
  keeps install, console, release, handoff graph, office registry, and office
  train helper payloads in a bounded SDK contract route.
