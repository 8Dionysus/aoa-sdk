# Antifragility Parts

Active SDK Antifragility parts live here. Each part is a route card plus the
payload it owns; sibling repositories keep stronger truth for proof, memory,
runtime response, owner-local repair, and deletion.

## Part Map

| Part | Role | Active surfaces |
| --- | --- | --- |
| [`stress-posture-dispatch-gate`](stress-posture-dispatch-gate/README.md) | Turns typed stress signals into narrowing-only SDK task-dispatch constraints. | docs, examples, tests |
| [`reviewed-stress-closeout-carry`](reviewed-stress-closeout-carry/README.md) | Carries reviewed stress evidence through closeout without promotion or repair authority. | docs, example manifest, tests |
| [`via-negativa`](via-negativa/README.md) | Keeps SDK pruning questions explicit before adding new helper surface. | checklist doc, test |

## Active Part Contract

Every active part keeps:

- `README.md`: role, input, output, owner split, next route
- `CONTRACT.md`: allowed outputs and stop-lines
- `VALIDATION.md`: narrow validation route

Old root paths and historical filenames route through parent provenance and
artifact topology, not through active compatibility aliases.
