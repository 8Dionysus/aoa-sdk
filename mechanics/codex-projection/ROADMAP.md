# Codex Projection Roadmap

This roadmap owns future pressure for the SDK Codex Projection mechanic.

## Current Contour

Codex Projection owns the workspace MCP server, live rollout status readouts,
portability boundaries, and owner rollout reference handoff. It projects
inspectable SDK context into Codex-facing routes without becoming session
memory or GitHub truth.

## Next Work

- Keep rollout readouts lower authority than raw session archives, GitHub
  state, and owner repositories.
- Keep portability boundaries explicit about what can travel and what remains
  local runtime context.
- Keep owner rollout references as handoff routes, not as proof that owner work
  landed.

## When Time Comes

- Add new projection routes only when a repeated Codex-facing read need cannot
  stay inside the current MCP, status, portability, or handoff parts.
- Add remote or richer MCP behavior only after local-first contracts remain
  stable.

## Out Of Scope

- session-memory authority;
- GitHub merge or CI truth;
- owner rollout acceptance;
- hidden Codex activation.

## Update Trigger

Update this roadmap when Codex-facing projection scope, portability posture,
status readout semantics, or future projection depth changes.
