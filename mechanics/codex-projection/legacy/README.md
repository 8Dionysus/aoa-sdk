# Codex Projection Legacy

Legacy preserves former Codex Projection mechanics parent names and migration
receipts. It is not an active route and must not become a compatibility
fallback.

## Layout

- `former-routes.json`: machine-readable former-name to active-route map.
- `INDEX.md`: human lookup from former name to active part.
- `DISTILLATION_LOG.md`: dated accounting for the correction.
- `raw/`: evidence-only district for future historical snapshots.

The active package reaches this district through
[`../PROVENANCE.md`](../PROVENANCE.md). Routine edits start from active Codex
Projection parts, not from legacy.

## Stop-lines

- Do not use legacy names in active route maps.
- Do not make `raw/` the only place a current rule exists.
- Do not hide new Codex-facing behavior in legacy.
