# Component Manifest Gate

## Use When

Use this part when recurrence component manifests, mixed adjacent manifests, or
manifest scan reports change.

## Do Not Use When

Do not use it to decide owner meaning or to coerce Agon-shaped manifests into
recurrence components.

## Route Check

Input is a manifest district. Output is a loaded component, foreign-manifest
notice, or diagnostic. The next route is graph closure, hook observation, or
owner repair.

## Active Outputs

- change signal schema and typed detection output
- manifest scan report
- component template example
- part-local recurrence component and adapter manifest examples
- compatibility diagnostics

## Next Route

Graph, hooks, live observations, and downstream projections consume only the
typed or diagnosed result of this gate.
