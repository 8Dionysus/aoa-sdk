# Wiring Rollout Handoff

## Use When

Use this part when recurrence needs propagation plans, connectivity diagnostics,
wiring snippets, rollout windows, or return handoff packets.

## Do Not Use When

Do not use it as an installer or repair loop.

## Route Check

Input is recurrence obligation evidence. Output is a reviewable plan, diagnostic,
wiring snippet, rollout bundle, or handoff.

## Active Outputs

- propagation plan schema and example
- connectivity gap report schema and example
- return handoff schema and example
- wiring plan and rollout bundle schemas and examples
- hook and CI templates

## Next Route

Owner repositories adopt snippets manually and keep their local truth.
