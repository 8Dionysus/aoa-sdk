# Owner Review Surface

## Use When

Use this part when recurrence beacon pressure must become an owner-facing review
queue, dossier packet, owner summary, or usage-gap report.

## Do Not Use When

Do not use it to accept, suppress, or close owner decisions.

## Route Check

Input is beacon pressure. Output is a review surface that still requires an
owner decision.

## Active Outputs

- review queue schema and examples
- candidate dossier schema and examples
- owner review summary schema and examples
- usage-gap report schema and examples

## Next Route

Review decision closure records the owner decision after review.
