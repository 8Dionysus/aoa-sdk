# Checkpoint Roadmap

This roadmap owns future pressure for the SDK Checkpoint mechanic.

## Current Contour

Checkpoint owns session-growth capture, dirty-gate hook posture, review-note
gates, reviewed session handoff runner behavior, child-task re-entry, and
reviewed closeout context carry.

Checkpoint packets make review and return easier. They do not become memory,
proof, progression, owner acceptance, or hidden workflow execution.

## Next Work

- Keep checkpoint notes session-local until a reviewed promotion route accepts
  them.
- Keep reviewed handoff runner output as operator-visible evidence, not as an
  automatic closeout result.
- Keep reviewed closeout carry fields advisory and owner-routed.
- Add stricter packet checks only after repeated checkpoint work proves stable
  fields.

## When Time Comes

- Add a new checkpoint part when a repeated checkpoint route cannot fit capture,
  handoff runner, re-entry, or reviewed carry.
- Promote checkpoint output toward durable memory only through the memory owner
  and reviewed intake route.

## Out Of Scope

- durable memory truth;
- proof verdicts;
- progression or quest acceptance;
- automatic closeout, harvest, push, merge, or release logic.

## Update Trigger

Update this roadmap when checkpoint carry meaning, handoff runner posture,
re-entry semantics, or future checkpoint promotion conditions change.
