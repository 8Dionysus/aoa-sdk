# Boundary Bridge Roadmap

This roadmap owns future pressure for the SDK Boundary Bridge mechanic.

## Current Contour

Boundary Bridge owns typed facades, consumed-surface posture gates, skill
runtime bridges, technique promotion readiness readers, and owner-layer signal
handoff. It keeps SDK handles useful without importing sibling meaning.

## Next Work

- Keep consumed sibling paths explicit and compatibility-checked before new
  readers become normal routes.
- Keep skill runtime reporting below `aoa-skills` meaning and host truth.
- Keep owner-layer signal detection additive, inspectable, and weaker than
  owner acceptance.
- Add new consumed surfaces only through a compatibility map, route card,
  source owner, and test.

## When Time Comes

- Add a new bridge part when repeated cross-owner pressure cannot stay inside
  compatibility, skill bridge, technique readiness, or owner-layer handoff.
- Widen facade behavior only after sibling source paths and fail-closed drift
  checks are stable.

## Out Of Scope

- sibling source truth;
- routing policy authority;
- skill execution meaning;
- proof, memory, role, stats, KAG, playbook, or runtime ownership.

## Update Trigger

Update this roadmap when bridge scope, consumed-surface posture, handoff
semantics, or future facade depth changes.
