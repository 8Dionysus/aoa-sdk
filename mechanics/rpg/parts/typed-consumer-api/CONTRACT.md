# Typed Consumer API Contract

## Allowed Outputs

- Typed Python models over consumed RPG surfaces.
- Compatibility readouts for expected owner transport paths.
- Helper methods that return stable read objects.
- Regression tests proving additive upstream fields remain tolerated.

## Stop-Lines

- Do not author RPG gameplay semantics.
- Do not mutate quest, progression, reputation, frontend, or runtime state.
- Do not collapse multi-axis progression into one score.
- Do not treat collection wrappers as stronger truth than owner item contracts.

## Owner Split

The SDK owns readable typed access. RPG doctrine, runtime projection,
frontend behavior, proof, quest state, progression, and owner acceptance stay
with stronger owners.
