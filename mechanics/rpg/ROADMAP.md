# RPG Roadmap

RPG is the SDK route for typed reads over RPG-shaped surfaces. It transports
source-owned paths and consumer contracts without becoming gameplay runtime,
inventory state, progression truth, or quest acceptance.

## Current Contour

- Keep typed consumer API and surface-path transport routed through active
  `parts/`.
- Keep RPG helpers as typed reads and route helpers.
- Keep surface-path transport explicit about source owner, consumed path, and
  what the SDK does not author.
- Keep richer RPG vocabulary out unless the game form improves routing or
  action without becoming decorative weight.

## Next Work

- Tighten transport contracts where repeated RPG consumption proves stable
  source paths and failure behavior.
- Keep generated companions derived from source routes and tests.
- Route gameplay, progression, ability, loadout, and quest meaning back to the
  owners that can accept it.

## When Time Comes

- Add a new RPG part when repeated RPG consumption cannot stay inside typed
  consumer API or surface-path transport.
- Add generated companions only after source routes and tests are stable.
- Widen helper behavior only after source owners name the meaning and the SDK
  can keep transport separate from gameplay truth.

## Out Of Scope

- Gameplay runtime.
- Inventory or progression truth.
- Quest acceptance.
- Character, ability, or loadout authority.
- Hidden ontology.
