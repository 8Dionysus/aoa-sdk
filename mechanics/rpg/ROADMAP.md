# RPG Roadmap

This roadmap owns future pressure for the SDK RPG mechanic.

## Current Contour

RPG owns the typed consumer API and surface-path transport helper posture. It
lets SDK consumers read RPG-shaped surfaces without becoming gameplay runtime
or progression authority.

## Next Work

- Keep RPG helpers as typed reads and route helpers.
- Keep surface-path transport explicit about source owner and consumed path.
- Add richer helpers only when the source owner has named the meaning and the
  SDK can keep transport separate from gameplay truth.

## When Time Comes

- Add a new RPG part only when repeated RPG consumption cannot stay inside
  typed consumer API or surface-path transport.
- Add generated companions only after source routes and tests are stable.

## Out Of Scope

- gameplay runtime;
- inventory or progression truth;
- quest acceptance;
- character, ability, or loadout authority.

## Update Trigger

Update this roadmap when RPG helper scope, transport posture, source-owner
split, or future RPG consumption depth changes.
